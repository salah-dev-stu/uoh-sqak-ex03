# ADR-007: uv as Mandatory Package Manager (pip/venv Forbidden)

**Status:** Accepted
**Date:** 2026-06-06
**Authors:** Salah Qadah, Andalus Kalash (pair uoh-sqak)
**Course:** 203.3763 — Orchestration of AI Agents, University of Haifa

---

## Context

The course grading rubric (Rule 12) explicitly forbids `pip`, `pip install`, `python -m`, `venv`, and `virtualenv`. All package operations must use `uv`. This is an automatic disqualification if violated, caught by automated scanning of `Makefile`, `README.md`, and CI workflows.

---

## Decision

Use **`uv`** (Astral) for all package management, virtual environment creation, script execution, and Python version pinning. No exceptions.

---

## Command Reference

| Operation | Correct (`uv`) | Forbidden |
|---|---|---|
| Install deps | `uv sync` | `pip install -r requirements.txt` |
| Add package | `uv add crewai` | `pip install crewai` |
| Run script | `uv run python -m agent_article.main` | `python main.py` |
| Run tests | `uv run pytest` | `pytest` |
| Run linter | `uv run ruff check src/` | `ruff check src/` |
| Pin Python | `uv python pin 3.13` | `pyenv local 3.13` |

---

## Rationale

1. **Rubric compliance**: Rule 12 is explicitly auto-scanned. Any `pip` or bare `python` invocation in shell scripts, Makefiles, or CI files is a deduction.
2. **Speed**: `uv` resolves and installs dependencies 10–100x faster than `pip` due to Rust-based resolver and parallel downloads. On the grader's machine, `uv sync` on a fresh clone completes in seconds.
3. **Lockfile reproducibility**: `uv.lock` pins every transitive dependency to an exact version. The grader's environment is byte-for-byte identical to the development environment. No "works on my machine" failures.
4. **Python version pinning**: `.python-version` file (generated by `uv python pin 3.13`) ensures the exact interpreter is used. TeXLive and LuaLaTeX are unaffected (they are system-level, not Python packages).
5. **`pyproject.toml` integration**: All project metadata, dependencies, ruff config, and pytest config live in `pyproject.toml`. `uv` reads this natively — no `setup.py`, `setup.cfg`, or `requirements.txt`.

---

## Project Configuration

```toml
# pyproject.toml (excerpt)
[project]
requires-python = ">=3.13"
dependencies = [
    "crewai>=0.28",
    "langchain-anthropic>=0.1",
    "matplotlib>=3.8",
    "rich>=13.0",
]

[tool.uv]
dev-dependencies = [
    "pytest>=8.0",
    "pytest-cov>=5.0",
    "ruff>=0.4",
]
```

---

## Consequences

- **Positive**: Deterministic builds; fast CI; no Python environment drift; grader can `uv sync` on any platform.
- **Positive**: `uv.lock` is committed to the repository — grader sees exact dependency graph.
- **Negative**: Grader must have `uv` installed. Mitigated by: `uv` is a single statically-linked binary; install instructions in README are one line: `curl -LsSf https://astral.sh/uv/install.sh | sh`.
- **CI**: GitHub Actions workflow uses `astral-sh/setup-uv@v3` action for zero-config uv installation.
