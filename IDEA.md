## The assignment in one sentence

Build, using **CrewAI**, a multi-agent team that writes a **15-page article/book** in **LaTeX** on a topic of your choice, with technical excellence in document production: linked bibliography, BiDi (Hebrew↔English), Python-generated charts, math formulas, TikZ diagrams.

### Content requirements (Section 13.1) — verbatim
1. **היקף: כ-15 עמודים (סוכם בכיתה; עברית קשה יותר ולכן מוערכת יותר)** = scope: ~15 pages (agreed in class; **Hebrew is harder and therefore more appreciated**)
2. **עמוד שער (Cover Sheet): נושא, שם הכותב, תאריך, קורס ומרצה** = Cover sheet: topic, author name, date, course, lecturer
3. **תוכן עניינים, חלוקה לפרקים, וכותרות עליונות/תחתונות (Footers/Headers)** = Table of contents, chapter division, headers/footers
4. **לפחות: תמונה אחת, גרף אחד שנוצר בקוד Python, טבלה אחת ונוסחה מתמטית אחת** = At least: 1 image, 1 graph **generated by Python code**, 1 table, 1 math formula
5. **שילוב עברית-אנגלית (BiDi) — לפחות פרק אחד שמדגים מעבר נכון בין ימין-לשמאל לשמאל-לימין** = Hebrew-English mix (BiDi) — at least one chapter demonstrating correct right-to-left ↔ left-to-right transitions
6. **רשימת ביבליוגרפיה בסוף, עם ציטוטים מקושרים** = Bibliography list at end, with linked citations

### Recommended technical workflow (Section 13.2) — verbatim
- *"מוסיפים לשרשרת CrewAI סוכן המייצר את קובצי ה-LaTeX. מומלץ לעבוד תחילה ב-Markdown (מהיר ונוח לבקרה), ורק כשהתוכן מושלם — להמיר ל-.tex ולקמפל."*
  → Add a LaTeX-producing agent to the CrewAI chain. Recommended: work first in **Markdown** (fast and easy to control), only convert to `.tex` when content is complete.
- **Compiler**: MiKTeX, recommended **LuaLaTeX** (good Hebrew support); **XeLaTeX** allowed.
- **Bibliography**: `.bib` files + **biblatex package + biber** compiler (NOT legacy `bibtex`; lecture L1524 names biber specifically).
- **Compilation passes**: when both `.tex` and `.bib` exist, **~4 compilations** are required for all references and citations to update. If clicking a reference doesn't jump to citation — missing compilation.
- **Graphics**: **TikZ** library for block diagrams.
- **Formulas**: math package; request *"fancy formula"* and not *"plain text"*. Sometimes due to Hebrew-English mix, the model outputs formula as flat text — then request fix.

### THE evaluation criterion — verbatim quote in the spec
> "הבדיקה היא טכנית, על המעטפת ולא על נכונות התוכן: שהקישורים מחוברים, הציטוטים קיימים, ה-BiDi תקין, הטבלאות אינן חורגות מהדף והנוסחאות מפוארות. שמרו את הפייפ הזה — הוא כלי ייצור מסמכים לכל מקום עבודה."
>
> *"The check is **technical, on the envelope and not on content correctness**: that links connect, citations exist, BiDi is correct, tables don't exceed the page, formulas are fancy. **Keep this pipeline — it's a document production tool for any workplace.**"*

This is the grading focus. **A beautiful, technically perfect 15-page LaTeX PDF on a moderately interesting topic outscores a brilliantly-written PDF with broken citations.**

## CrewAI mandatory architecture

Per HW3 spec + Appendix A of the L06 summary + Lec 06 PDFs:

### Roles (at minimum 4 agents; add more for bonus)
1. **Researcher Agent** — finds sources, organizes notes
   - Skill: `researcher_skill/` with checklist + reference format
   - Tools: web_search, file_read
2. **Writer Agent** — turns research into prose (Hebrew preferred)
   - Skill: `writer_skill/` with style guide + tone instructions
   - Tools: cite_lookup, markdown_writer
3. **Editor Agent** — polishes, catches errors, verifies citations
   - Skill: `editor_skill/` with style + grammar checklist
   - Tools: text_diff, citation_checker
4. **LaTeX-Producer Agent** — converts approved Markdown to .tex, handles BiDi, fancy formulas, TikZ
   - Skill: `latex_skill/` with template + BiDi recipe + formula formatting rules
   - Tools: latex_compile, biber_compile, chart_generator (matplotlib→PNG)

### Optional bonus agents
- **Chart Designer Agent** — Python code that produces matplotlib charts
- **Bibliography Curator Agent** — manages `.bib` entries with consistent format
- **Style Reviewer Agent** — runs after compile to verify "envelope" criteria (links, BiDi, formula format)

### Process
CrewAI supports `Process.sequential` and `Process.hierarchical`. Sequential is simpler; hierarchical lets one "manager agent" delegate. **Default: sequential** (Researcher → Writer → Editor → LaTeX-Producer). Hierarchical mode is bonus.

### Skill layer (from Spec Appendix A)
Each agent gets a `Skill` directory:
```
src/<package>/skills/
├── researcher_skill/
│   ├── SKILL.md          # YAML frontmatter (name, description) + Markdown body
│   ├── references/        # checklists, reference style guides
│   └── scripts/           # optional helpers
├── writer_skill/...
├── editor_skill/...
└── latex_skill/...
```

`SKILL.md` format (from spec):
```markdown
---
name: code-review
description: Senior code reviewer; checks for security, performance, style
---
You are an expert code reviewer. When you see code:
1. Check for SQL injection, XSS, secrets...
2. ...
```

**Skill vs Tool distinction** (Dr. Segal's verbatim analogy):
- **Tool** = the screwdriver / drill / search engine the worker holds in his hand. Action capability ("the what").
- **Skill** = the onboarding packet ("the how"). The procedure manual, checklists, style guide. A professional worker needs BOTH.

## What you must NOT do

- ❌ Submit a PDF with broken citation links (auto-fails the "envelope" check)
- ❌ Use plain-text formulas instead of "fancy" math typesetting (the LaTeX-Producer agent prompt MUST contain the literal English phrase **"fancy formula, not plain text"** per L1742–1748)
- ❌ Hardcode the article content (it must come from real Crew agent runs)
- ❌ Skip the Skill layer (spec Appendix A explicitly defines this as the HW3 expertise pattern)
- ❌ Push code without including `latex/` source directory (must be committed)
- ❌ Submit just a PDF with no source — the LaTeX project and Python source must both be in the repo
- ❌ Use Arabic in the output (Hebrew or English; Dr. Segal wants to read it)
- ❌ **Use Overleaf** (L1622: lecturer asked, *"לא, לא, ממש לא. אני מתעקש על זה"* — "No, really not. I insist."). Local LaTeX install + Python-driven compile is the only acceptable shape.
- ❌ **Skip the Crew architecture diagram inside the article** (L1708–1714: TikZ block diagram of your CrewAI architecture must appear in the article. The OOP class diagram is the "extra mile" bonus path, *"מי שמגדיל ראש"*.)
- ❌ Use legacy `bibtex` — must be **biblatex package + biber** compiler (L1524 explicit).
- ❌ Skip the 4th `lualatex` pass — citations and ToC won't resolve. Makefile must do exactly **lualatex → biber → lualatex → lualatex**.
