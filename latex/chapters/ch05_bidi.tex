\section[\texthebrew{מהי דו-כיוניות?}]{\texthebrew{מהי דו-כיוניות?}\mbox{\setLTR\enspace\thesection}}
\label{sec:bidi-meaning}

\begin{hebrew}
המושג דו-כיוניות, המוכר בשמו הלועזי \foreignlanguage{english}{BiDi}
(ראשי תיבות של \foreignlanguage{english}{Bidirectional Text}),
מתאר את היכולת של מערכת עיבוד טקסט להציג תוכן הכתוב בכיוונים שונים
בתוך אותו מסמך.
בשפות שמיות כגון עברית וערבית הכתיבה מתנהלת מימין לשמאל
(\foreignlanguage{english}{RTL}),
ואילו בשפות לטיניות כמו אנגלית, גרמנית וצרפתית הכתיבה מתנהלת משמאל לימין
(\foreignlanguage{english}{LTR}).
כאשר מסמך מדעי, אקדמי או טכני מכיל את שתי השפות זו לצד זו, על מנגנון
הדו-כיוניות לקבוע בצורה נכונה את כיוון הטקסט לכל קטע, לכל שורה, ולכל תו.

הסטנדרט הבינלאומי המגדיר את כללי הדו-כיוניות הוא
\foreignlanguage{english}{Unicode} \mbox{\cite{Unicode2023}}.
האלגוריתם \foreignlanguage{english}{Unicode BiDi} מסווג כל תו לאחת
מקטגוריות כיוון: תווים בעלי כיוון חזק (אותיות עבריות ואותיות לטיניות),
תווים בעלי כיוון חלש (ספרות ערביות וסימנים מתמטיים),
ותווים ניטרליים (רווחים וסימני פיסוק).
הסדר הלוגי של התווים—זה השמור בקובץ הטקסט כפי שנכתב על-ידי הכותב—שונה מהסדר
הוויזואלי המוצג על המסך לקורא.
האלגוריתם משמש גשר בין שני הסדרים הללו, ומבטיח שהתוצאה הוויזואלית תואמת
את כוונת הכותב.

במסגרת מערכות סוכני בינה מלאכותית \mbox{\cite{Segal2026}},
כאשר סוכן מייצר מסמכי \foreignlanguage{english}{LaTeX} המשלבים עברית ואנגלית,
עליו לשלב את כללי הדו-כיוניות בתפוקתו.
כשל בהטמעת הכללים מוביל לתצוגה הפוכה של מספרי ציטוטים, להפיכת שמות פקודות
לטיניות, ולעיצוב שגוי של כותרות פרקים וסעיפים.
\end{hebrew}

\section[\texthebrew{פתרון \foreignlanguage{english}{LuaLaTeX} ו-\foreignlanguage{english}{Polyglossia}}]{\texthebrew{פתרון \foreignlanguage{english}{LuaLaTeX} ו-\foreignlanguage{english}{Polyglossia}}\mbox{\setLTR\enspace\thesection}}
\label{sec:bidi-solution}

\begin{hebrew}
הפתרון המומלץ למסמכי \foreignlanguage{english}{LaTeX} דו-כיוניים הוא
שילוב מהדר \foreignlanguage{english}{LuaLaTeX} עם חבילת
\foreignlanguage{english}{Polyglossia} \mbox{\cite{Polyglossia2023}}.
חבילה זו, שפותחה כממשיכה המודרנית של \foreignlanguage{english}{Babel},
מספקת תמיכה מקיפה בעשרות שפות—כולל עברית, ערבית, יוונית ושפות אסיאתיות—
תוך שימוש בגופנים \foreignlanguage{english}{OpenType} ובמנגנון כיוון המבוסס
על \foreignlanguage{english}{Unicode}.

הגדרת השפה הראשית והשפה הנוספת מתבצעת בקדמת המסמך (הפריאמבל):
הפקודה \foreignlanguage{english}{\texttt{\textbackslash setmainlanguage\{hebrew\}}} קובעת את העברית
כשפה הראשית, ו-\foreignlanguage{english}{\texttt{\textbackslash setotherlanguage\{english\}}} מגדירה
את האנגלית כשפה משנית.
לאחר הגדרות אלה, כל הטקסט ב-\foreignlanguage{english}{RTL} כברירת מחדל,
ומעברים לאנגלית מתבצעים על-ידי עטיפת הקטע האנגלי בפקודה
\foreignlanguage{english}{\texttt{\textbackslash foreignlanguage\{english\}\{\ldots\}}}
או על-ידי שימוש בסביבה
\foreignlanguage{english}{\texttt{\textbackslash begin\{english\}\ldots\textbackslash end\{english\}}}.

מהדר \foreignlanguage{english}{LuaLaTeX} כולל אוטומטית את ספריית
\foreignlanguage{english}{luabidi}, המטפלת בהיפוך הכיוון ברמת הפרגרף.
שילוב זה מבטיח שסדר הכיוון מנוהל נכון גם כאשר פרגרף אחד מכיל מספר מעברים
בין כיוונים שונים \mbox{\cite{Polyglossia2023}}.

שני כללים קריטיים שסוכן \foreignlanguage{english}{AI} המייצר
\foreignlanguage{english}{LaTeX} חייב ליישם ללא יוצא מן הכלל:
ראשית, כל מילה אנגלית בתוך פקודת
\foreignlanguage{english}{\texttt{\textbackslash texthebrew\{\ldots\}}}
חייבת להיות עטופה ב-\foreignlanguage{english}{\texttt{\textbackslash foreignlanguage\{english\}\{\ldots\}}},
אחרת \foreignlanguage{english}{LuaLaTeX} עלול להפוך את סדר האותיות בשם
הפקודה הלטיני.
שנית, כל פקודת \foreignlanguage{english}{\texttt{\textbackslash cite\{\}}} בתוך
\foreignlanguage{english}{\texttt{\textbackslash begin\{hebrew\}\ldots\textbackslash end\{hebrew\}}}
חייבת להיות עטופה ב-\foreignlanguage{english}{\texttt{\textbackslash mbox\{\ldots\}}},
שכן ללא עטיפה זו המספר הרב-ספרתי
\foreignlanguage{english}{[10]} יוצג כ-\foreignlanguage{english}{[01]}.
\end{hebrew}

\section[\texthebrew{אתגרים בסביבת ייצור}]{\texthebrew{אתגרים בסביבת ייצור}\mbox{\setLTR\enspace\thesection}}
\label{sec:bidi-production}

\begin{hebrew}
בפריסה של מערכת סוכנים מבוססת \foreignlanguage{english}{CrewAI}
\mbox{\cite{CrewAI2024}} שמשימתה לייצר מסמכי \foreignlanguage{english}{LaTeX}
דו-לשוניים, עולים שלושה אתגרים מרכזיים בסביבת הייצור.

\textbf{עקביות כללי הכיוון.}
מודל שפה גדול (\foreignlanguage{english}{LLM}) עשוי לייצר תוכן שבו
מילים אנגליות בתוך פקודת \foreignlanguage{english}{\texttt{\textbackslash texthebrew\{\}}}
אינן עטופות בפקודת הכיוון המתאימה, ופקודות ציטוט אינן מוגנות
ב-\foreignlanguage{english}{\texttt{\textbackslash mbox\{\}}}.
הפתרון הוא לשלב ב-\foreignlanguage{english}{SKILL.md} של הסוכן המייצר
\foreignlanguage{english}{LaTeX} רשימה מפורשת של הכללים, דוגמאות נכונות
ודוגמאות שגויות, וכלל ברור של "אין חריגות"
\mbox{\cite{Anthropic2024b}}.

\textbf{גלישת טבלאות.}
כאשר תאי טבלה מכילים טקסט ארוך בשתי שפות, קיים סיכון ממשי שהטבלה תגלוש
מעבר לשולי העמוד.
הפתרון המוכח הוא לדרוש מהסוכן להשתמש בסוג עמודה
\texttt{\foreignlanguage{english}{p\{Xcm\}}} בלבד (עם רוחב מפורש בסנטימטרים),
ולהימנע לחלוטין מסוגי העמודות
\texttt{\foreignlanguage{english}{l}},
\texttt{\foreignlanguage{english}{r}} ו-\texttt{\foreignlanguage{english}{c}},
שאינם מגבילים את רוחב התא ואינם עוצרים גלישה.

\textbf{ניהול מרדפי הידור.}
מסמך הכולל ביבליוגרפיה בפורמט \foreignlanguage{english}{BibLaTeX} עם
מנהל \foreignlanguage{english}{biber} ודו-כיוניות מחייב לפחות ארבעה מרדפי
\foreignlanguage{english}{LuaLaTeX} ומרדף \foreignlanguage{english}{biber}
אחד ביניהם, על-מנת שכל ההפניות הצולבות, הציטוטים ונתוני הכותרות יתייצבו
כראוי.
\foreignlanguage{english}{Makefile} המכיל את כל המרדפים הנדרשים מונע שגיאות
של "מרדף חסר" בסביבת שילוב רציף (\foreignlanguage{english}{CI}).
\end{hebrew}

\section{Mixed-Language Example}
\label{sec:bidi-mixed}

\texttt{luabidi} command set applied throughout Sections~5.1--5.3.
Table~\ref{tab:bidi-commands} catalogues the essential macros, their descriptions,
and the contexts in which they must be used.
Every command listed was embedded into the \emph{LaTeX Producer} agent's
\texttt{SKILL.md} file within the \textsc{CrewAI} pipeline
(see Chapter~\ref{ch:casestudy}),
so that agent-generated source compiles cleanly on the first pass
without manual post-processing.

Adherence to these rules was enforced programmatically: the \emph{Editor} agent
ran a post-processing verification pass on every generated \texttt{.tex} file,
flagging any English word inside a \texttt{\textbackslash texthebrew\{\}} argument
that was not wrapped in
\texttt{\textbackslash foreignlanguage\{english\}\{\ldots\}},
and any \texttt{\textbackslash cite} inside a \texttt{hebrew} environment lacking
an enclosing \texttt{\textbackslash mbox}.
Only source files passing both checks advanced to the compilation stage,
ensuring that the final PDF was free of BiDi artefacts.
