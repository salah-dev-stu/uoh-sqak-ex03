\chapter{Orchestration Frameworks}\label{ch:frameworks}

Agent orchestration requires selecting among competing frameworks, each optimizing for different architectural priorities. Three frameworks dominate contemporary multi-agent application development: LangChain provides modular infrastructure for language model application construction; CrewAI delivers role-based agent teams with built-in process abstractions; LangGraph enables stateful graph-based orchestration with conditional routing and human-in-the-loop integration points. Understanding their distinct strengths, architectural philosophies, and production readiness profiles enables architects to select tools matching problem characteristics and organizational constraints \cite{Segal2026}.

\section{LangChain and LCEL: Modular Component Framework}

LangChain operationalizes a foundational principle Dr. Segal articulates: ``a standalone language model is just one component in a system'' \cite{Segal2026}. Rather than prescribing agent patterns directly, LangChain supplies reusable building blocks---Document Loaders, Text Splitters, Embeddings, Vector Stores, Prompt Templates, Output Parsers, and Retrievers---that engineers combine to construct retrieval-augmented generation (RAG) pipelines, chains, and agents \cite{LangChain2024a}.

The LangChain Expression Language (LCEL) treats each component as a composable object with a standard interface. This design enables modular architecture: swap the LLM provider from OpenAI to Anthropic to open-source models through a single parameter change; replace the vector store from Chroma to FAISS without rewriting retrieval logic; insert intermediate processing steps into chains by composing objects with the pipe operator. LCEL supports both synchronous and asynchronous execution, streaming, and batch processing from identical code \cite{LangChain2024a}.

LangChain's strength emerges in infrastructure-heavy applications where component modularity and library ecosystem depth matter. A RAG system built with LangChain leverages Document Loaders optimized for PDFs, databases, web APIs, and specialized formats; Text Splitters that preserve semantic boundaries; Embeddings integrated with commercial and open-source models; Vector Stores with built-in similarity search and metadata filtering. This depth reduces engineering effort substantially compared to building retrieval infrastructure from scratch.

The tradeoff is engineering responsibility. LangChain provides components but requires developers to orchestrate them---define the flow, handle failures, manage context windows, optimize token consumption, and ensure output quality. Production systems built with LangChain often wrap core chains in guardrails, observability layers, and evaluation frameworks.

\section{CrewAI: Role-Based Agent Teams}

CrewAI addresses the orchestration challenge by providing high-level abstractions specifically designed for multi-agent workflows. The framework models systems as teams of specialized agents, each with a defined role, goal, and backstory. Agents collaborate through tasks---units of work that specify what is required, which agent executes it, and what tools are available. CrewAI handles context passing automatically: the output from one agent becomes the input for the next without manual copying, reducing token consumption by 40--60\% compared to naive agent chaining \cite{CrewAI2024}.

CrewAI's \texttt{Process} abstraction enables two distinct collaboration topologies. Sequential process assumes a fixed execution order: the Researcher completes their task, the Writer receives the research as context, the Editor refines the output, and the Formatter prepares the final artifact. Hierarchical process introduces a Manager agent that views the overall goal, creates a plan, coordinates agents, and makes dynamic decisions---looping back to the Researcher if the Writer identifies information gaps \cite{Segal2026}.

Content generation pipelines map naturally to CrewAI teams. An article-writing system decomposes into four roles: Researcher (discovers information), Writer (composes narrative), Editor (refines clarity), Formatter (adjusts layout for publication). This role-to-agent mapping is intuitive and requires minimal orchestration logic. The framework automates context threading, agent sequencing, and output aggregation.

CrewAI excels when problems decompose cleanly into specialized roles with clear handoff points. The tradeoff is reduced flexibility for non-hierarchical, graph-based workflows where execution paths branch conditionally or require exploratory loops. CrewAI's process abstractions assume relatively linear topologies.

\section{LangGraph: Stateful Graph-Based Orchestration}

LangGraph addresses tasks that require non-predetermined execution paths, loops, branches, and conditional routing. While CrewAI assumes a fixed process topology, LangGraph enables arbitrary directed acyclic graphs (DAGs) where nodes represent agent invocations or tool calls and edges represent transitions controlled by conditional logic or human approval \cite{Segal2026}.

Consider a customer support system that classifies incoming requests as bug reports, feature requests, or billing inquiries, then routes each to a specialized handler. The execution path depends on classification output. With CrewAI's sequential process, this would require a router agent as the first step, followed by conditional task assignment---feasible but awkward. LangGraph models this as a graph: a classification node branches to three distinct handler subgraphs based on output.

LangGraph's native human-in-the-loop capability addresses a critical production requirement. Agents can pause at decision points, returning intermediate state to a human operator for review and approval. After human approval, execution resumes from that checkpoint. This prevents closed-loop autonomous agent operation and is essential for systems handling sensitive actions---approving transactions, deleting data, or modifying permissions. CrewAI supports task-level checkpoints but requires custom implementation; LangGraph provides native state inspection and resumption.

The framework models state explicitly as a dictionary of values that transitions modify. This clarity simplifies reasoning about what information agents can access at each step and enables deterministic replay of executions for debugging and audit trails.

LangGraph's tradeoff is steeper learning curve. State machine semantics and graph definition syntax require developers to think in terms of nodes, edges, and conditional branching---a departure from the agent/task/crew abstractions of CrewAI \cite{Segal2026}.

\section{Microsoft AutoGen: Multi-Turn Conversations and Group Chat}

Microsoft's AutoGen framework orchestrates multi-agent systems through a conversation and chat-based metaphor \cite{AutoGen2023}. Rather than explicit task assignment, agents participate in a group chat where each agent sees the conversation history and can decide whether to respond. An Agent has a system message defining its role, a reasoning module, and optionally tool access.

AutoGen excels in scenarios where agent coordination emerges through natural dialogue. A brainstorming session with multiple agents, each contributing ideas, works naturally as a group chat. Agents can ask clarifying questions, challenge proposals, and refine ideas iteratively \cite{AutoGen2023}.

The framework provides flexible termination conditions: a conversation ends when a designated agent approves the output, when maximum turns are reached, or when a termination function evaluates current state. This enables workflows where agreement or consensus matters---not just sequential hand-offs.

AutoGen's tradeoff is higher token consumption (all agents see full conversation history) and less deterministic routing (agents decide independently when to speak rather than being explicitly invoked) \cite{Segal2026}.

Selection among these frameworks depends on problem structure, team expertise, and production constraints. CrewAI suits well-decomposed workflows with clear role boundaries and sequential or hierarchical execution. The framework's built-in process abstractions and context threading eliminate orchestration boilerplate. LangChain provides maximum flexibility and ecosystem depth at the cost of engineering responsibility; teams build their own abstractions and guardrails. LangGraph enables non-linear workflows with conditional branching and human approval gates essential for production safety \cite{Segal2026}.

Many production systems combine frameworks: CrewAI agents invoke LangChain tools for retrieval and data processing, which may trigger LangGraph subworkflows for exploratory reasoning with embedded approval checkpoints. This composition strategy balances ease of use from CrewAI's abstractions with flexibility from LangChain's components and safety from LangGraph's state machines. Architects designing multi-agent systems should evaluate these frameworks not as mutually exclusive choices but as complementary tools addressing distinct layers of orchestration complexity \cite{Segal2026, CrewAI2024, LangChain2024a}.
