\chapter{Orchestration Frameworks}\label{ch:frameworks}

The landscape of multi-agent orchestration frameworks has matured significantly, with multiple mature, production-ready options available to practitioners. Each framework embodies different design philosophies: CrewAI emphasizes role-based agents and structured processes; LangChain provides modular, composable primitives; LangGraph introduces explicit state machines; and Microsoft AutoGen focuses on conversational collaboration. This chapter surveys the four dominant frameworks and provides comparative analysis to guide architectural selection.

\section{LangChain and LCEL}

LangChain, developed by Anthropic and the open-source community, pioneered the modular approach to agent orchestration. Rather than impose a specific architecture, LangChain provides low-level building blocks: Language Models (LMs), chains, memory abstractions, tools, and agent implementations. The core abstraction is the \texttt{Runnable}, a composable interface that accepts input and produces output. Chains are composed by piping Runnables together; LangChain Expression Language (LCEL) provides a concise syntax for chain composition.

\cite{LangChain2024a} documents the LCEL paradigm: chains are declaratively defined by composing primitives, enabling introspection, serialization, and batch processing. A research chain, for example, pipes a query router through a search tool, then through an LLM that synthesizes results. LangChain's strength lies in flexibility---engineers can build arbitrary topologies without framework constraints. The trade-off is verbosity: even simple chains require explicit definition of routing logic, error handling, and state management.

LangChain agents use the ReAct (Reasoning \& Acting) pattern, where an agent reasons about a problem, selects a tool, observes the result, and repeats until a stopping condition is met. This pattern is well-suited to open-ended problem solving but requires careful tuning of stopping criteria and token budgets to prevent runaway loops.

\section{CrewAI Role-Based Agents}

CrewAI, introduced in early 2024, takes a deliberate stance: agents are defined by role, goal, and backstory. A task specifies what work the agent performs and on what input. Processes (sequential, hierarchical, or custom callbacks) determine execution order. This opinionated design accelerates development---the framework provides sensible defaults for agent prompting, tool binding, and inter-agent communication.

\cite{CrewAI2024} emphasizes practical production patterns. Each agent's role is a natural language description of expertise; goal describes the desired outcome; backstory provides context that grounds the agent's reasoning. Tasks are defined declaratively, then bound to a Crew (a collection of agents). The framework automatically handles prompt assembly, tool invocation, and output validation. CrewAI's strength is rapid iteration: complex multi-agent pipelines can be prototyped in minutes. The trade-off is rigidity---custom topologies require callback-based process overrides, and debugging relies on log analysis rather than code introspection.

\section{LangGraph Stateful Graphs}

LangGraph, developed by Anthropic, models agent orchestration as an explicit state machine represented as a directed acyclic graph (DAG). Nodes represent agents, decision points, or compute steps; edges represent state transitions and data flow. LangGraph provides explicit primitives for conditional branching, loops, synchronization, and error recovery.

\cite{LangGraph2024} details the state machine approach. A graph is defined by initial state, a set of node functions, and transition rules. Node functions receive the current state, perform computation, and return updated state. Conditional edges allow routing based on state inspection---if a Researcher agent produces low-confidence output, execution can loop back for refinement. This explicitness enables strong guarantees: deterministic replay, comprehensive error traces, and precise control over execution flow.

LangGraph's strength lies in determinism and observability---every state transition is recorded and can be replayed. The trade-off is complexity: engineers must model their workflow as an explicit state machine, requiring more upfront design than CrewAI's implicit approaches. However, for mission-critical systems where reproducibility and auditability are paramount, this investment pays dividends.

\section{Microsoft AutoGen}

Microsoft's AutoGen framework prioritizes multi-agent conversation and collaborative reasoning. Agents communicate via multi-turn dialogue; a conversation manager orchestrates exchanges. AutoGen is purpose-built for debate, brainstorming, and iterative refinement scenarios where agent interaction patterns are not predetermined.

\cite{AutoGen2023} describes the conversation-based model: agents receive the conversation history, generate responses, and participate in dialogue. The conversation manager monitors dialogue quality and can inject guidance, terminate loops, or escalate to human oversight. AutoGen's strength lies in emergence---collaborative problem solving can spontaneously improve as agents interact. The trade-off is unpredictability: conversation-based systems offer no guarantees about termination, latency, or output quality. Production deployments of AutoGen typically layer explicit supervision to bound conversation length and introduce deterministic termination.

\section{Selection Heuristics}

For teams seeking rapid time-to-market with sequential or hierarchical workflows, \textit{CrewAI} minimizes boilerplate and cognitive load. For integration into existing codebases or custom topologies requiring fine-grained control, \textit{LangChain} provides the necessary abstraction level. For production systems demanding reproducibility, auditability, and complex state management, \textit{LangGraph} offers explicit guarantees. For collaborative reasoning and debate scenarios, \textit{AutoGen} is purpose-built.

In practice, production deployments often combine frameworks: use CrewAI for agent definition and task specification, LangChain for tool binding and chain composition, and LangGraph for orchestration control at the top level. This layered approach captures the strengths of each framework while mitigating individual weaknesses.
