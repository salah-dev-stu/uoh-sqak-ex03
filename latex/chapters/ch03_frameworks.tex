\chapter{Orchestration Frameworks}
\label{ch:frameworks}

The landscape of multi-agent orchestration frameworks has matured rapidly since 2023.
Three frameworks dominate production deployments: \textbf{CrewAI}, \textbf{LangGraph}, and
\textbf{AutoGen}. Each embeds a distinct philosophy about how agents should be composed,
how state should flow, and how failures should be handled \cite{Segal2026}.

\section{CrewAI}

\textbf{CrewAI} models multi-agent collaboration as a \emph{crew} of role-playing agents
working toward a shared goal \cite{CrewAI2024}. Each agent is assigned a \texttt{role},
a \texttt{goal}, and a \texttt{backstory}---three prompt-level primitives that shape the
agent's persona and decision-making heuristics. Tasks are defined separately from agents
and assembled into a \texttt{Process} (sequential or hierarchical) at crew-assembly time.

The framework supports two execution modes:
\begin{itemize}
  \item \texttt{Process.sequential} --- agents execute in declaration order; each agent
        receives the previous agent's output as context.
  \item \texttt{Process.hierarchical} --- a manager LLM decomposes the goal, dispatches
        subtasks to workers, and synthesises results.
\end{itemize}

CrewAI's skills layer augments each agent with curated domain knowledge injected at
prompt-construction time, reducing hallucination on specialised topics \cite{CrewAI2024}.

\subsection{Token Cost Model for a CrewAI Crew}

For a crew of $K$ agents processing a task, the total inference cost $C_{\mathrm{crew}}$ is

\begin{equation}
  C_{\mathrm{crew}} = \sum_{k=1}^{K}\!\Bigl(
    c_{\mathrm{in}}\,t_{k}^{\mathrm{in}} + c_{\mathrm{out}}\,t_{k}^{\mathrm{out}}
  \Bigr) + C_{\mathrm{mgr}}
  \label{eq:crewai_cost}
\end{equation}

where $c_{\mathrm{in}}$ and $c_{\mathrm{out}}$ are the per-token input and output prices,
$t_{k}^{\mathrm{in}}$ and $t_{k}^{\mathrm{out}}$ are the token counts for agent~$k$, and
$C_{\mathrm{mgr}}$ is the manager agent's overhead cost in hierarchical mode.

\section{LangGraph}

\textbf{LangGraph} extends LangChain with a graph-based execution model in which nodes
represent arbitrary computation units (LLM calls, tool invocations, Python functions) and
edges encode conditional routing logic \cite{LangGraph2024}. State is a typed dictionary
that flows through the graph; every node reads from and writes to this shared state object.

The framework is particularly suitable for:
\begin{itemize}
  \item \textbf{Cyclic workflows} --- loops, retries, and human-in-the-loop approval gates
        are modelled as back-edges in the graph.
  \item \textbf{Conditional branching} --- a routing node inspects state and returns the
        name of the next node to execute, enabling dynamic task allocation.
  \item \textbf{Streaming} --- incremental state updates are streamed to the caller,
        enabling real-time progress monitoring.
\end{itemize}

\subsection{Graph Traversal Complexity}

For a directed acyclic task graph with $V$ nodes and $E$ conditional edges, the worst-case
number of LLM calls $\mathcal{N}$ satisfies

\begin{equation}
  \mathcal{N} \leq V + \sum_{e=1}^{E} r_e
  \label{eq:langgraph_calls}
\end{equation}

where $r_e \in \{0, 1\}$ indicates whether conditional edge~$e$ triggers a re-evaluation
call. In practice, well-designed graphs keep $\mathcal{N}$ close to $V$ by batching
routing decisions \cite{LangGraph2024}.

\section{AutoGen}

\textbf{AutoGen} from Microsoft Research frames multi-agent interaction as a
\emph{conversation} between \texttt{ConversableAgent} instances \cite{AutoGen2023}.
Agents exchange messages in a shared chat history; each agent decides autonomously
whether to reply, delegate, or terminate. This conversational abstraction lowers the
conceptual barrier for developers familiar with chat-completion APIs.

AutoGen's \texttt{GroupChat} mechanism supports $N$ agents in a round-robin or
LLM-directed speaker-selection protocol. The selection probability for agent~$j$ under
the LLM-directed policy is a function of conversation history $\mathcal{H}$:

\begin{equation}
  P(\text{speaker} = j \mid \mathcal{H}) =
  \frac{\exp\bigl(s_j(\mathcal{H})\bigr)}
       {\displaystyle\sum_{i=1}^{N} \exp\bigl(s_i(\mathcal{H})\bigr)}
  \label{eq:autogen_selection}
\end{equation}

where $s_j(\mathcal{H})$ is the relevance score assigned by the moderator LLM to
agent~$j$ given the conversation so far \cite{AutoGen2023}.

\section{LangChain and the Ecosystem}

\textbf{LangChain} provides the foundational primitives---prompt templates, output parsers,
memory stores, and tool abstractions---on which both LangGraph and many CrewAI integrations
are built \cite{LangGraph2024}. Its \texttt{Runnable} interface unifies chains, agents, and
tools under a single composable protocol, enabling lazy evaluation and streaming by default.

The broader ecosystem includes specialised components:
\begin{itemize}
  \item \textbf{LangSmith} --- observability and tracing platform for LangChain/LangGraph
        pipelines; captures every token, latency, and intermediate state.
  \item \textbf{LangServe} --- deploys any \texttt{Runnable} as a REST API with
        auto-generated OpenAPI documentation.
  \item \textbf{CrewAI Tools} --- a curated registry of plug-and-play tools (web search,
        file I/O, code execution) designed for CrewAI agent assignment.
\end{itemize}

\section{Selecting a Framework}

The choice among frameworks depends on the structure of the target task:

\begin{itemize}
  \item Choose \textbf{CrewAI} when the workflow maps cleanly to role-based delegation and
        when developer productivity and a low learning curve are priorities.
  \item Choose \textbf{LangGraph} when the workflow contains cycles, conditional retries,
        or requires fine-grained streaming observability \cite{LangGraph2024}.
  \item Choose \textbf{AutoGen} when the problem naturally decomposes into a multi-party
        dialogue and when Microsoft Azure integration is a requirement \cite{AutoGen2023}.
\end{itemize}

For the \texttt{uoh-sqak} article-generation system, CrewAI is selected as the primary
orchestration layer because its role-based model maps directly onto the editorial pipeline:
Researcher~$\to$ Writer~$\to$ Editor~$\to$ \LaTeX{} Producer \cite{CrewAI2024,Segal2026}.
LangGraph's graph primitives are retained as a secondary pattern for the retry loop
around the \LaTeX{} compilation step.

\section{Framework Performance Benchmarks}

Empirical benchmarks across 50 article-generation runs reveal the latency distribution
shown in Figure~\ref{fig:framework_perf}. The mean end-to-end latency $\bar{L}$ and
standard deviation $\sigma_L$ satisfy

\begin{equation}
  \bar{L} = \frac{1}{n}\sum_{i=1}^{n} L_i, \qquad
  \sigma_L = \sqrt{\frac{1}{n-1}\sum_{i=1}^{n}(L_i - \bar{L})^2}
  \label{eq:latency_stats}
\end{equation}

where $L_i$ is the wall-clock latency of the $i$-th run and $n = 50$ \cite{Segal2026}.
