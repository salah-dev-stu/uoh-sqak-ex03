\chapter{Orchestration Frameworks}\label{ch:frameworks}

\section{Framework Landscape}

Three frameworks dominate the multi-agent orchestration landscape: CrewAI, LangChain, and LangGraph. Each reflects different design philosophies\cite{CrewAI2024}. CrewAI prioritizes prescriptive patterns and ease-of-use for standard workflows. LangChain emphasizes composability and flexibility through LCEL (LangChain Expression Language). LangGraph introduces explicit state management and graph-based execution models. Understanding the trade-offs between these frameworks is essential for selecting the right tool for a given problem.

Additionally, Microsoft AutoGen provides a conversation-based approach where agents exchange messages in turns, offering a different perspective on multi-agent coordination\cite{AutoGen2023}. Each framework addresses core challenges in orchestration: task sequencing, state persistence, parallelism, tool integration, and observability.

\section{LangChain and LCEL}

LangChain pioneered composable building blocks for language model applications\cite{LangChain2024a}. At its core is the LCEL (LangChain Expression Language), a declarative syntax for combining chains, prompts, and tools into complex pipelines. A typical LCEL chain connects a prompt template to a model, then routes output through parsing and tool invocation.

LangChain's strength lies in flexibility: developers compose custom agents using lower-level primitives (agent executors, tool loops, memory strategies). This approach accommodates novel orchestration patterns and domain-specific requirements. For example, a research system might implement a web-search loop with semantic caching and hierarchical summarization—patterns not hard-coded into the framework.

The trade-off is increased architectural responsibility. Developers must design their own agent loops and state transitions, demanding expertise in multi-agent systems. Memory management requires selecting and configuring strategies (buffer, summary, vector store) appropriate for the use case. LangChain excels for custom and innovative workflows but steeper learning curves for standard patterns.

\section{CrewAI Role-Based Agents}

CrewAI adopts a prescriptive approach centered on agent teams. Each agent is defined by three attributes: \texttt{role} (job title), \texttt{goal} (what the agent aims to achieve), and \texttt{backstory} (persona and expertise). Teams are assembled with a process type (sequential, hierarchical, or consensus), and tasks are assigned to agents\cite{CrewAI2024}.

This model is intuitive for research-writing-editing pipelines. A Researcher agent gathers information, a Writer agent produces narrative, an Editor agent refines prose, and a LaTeX Producer agent formats output. Task definitions specify input requirements, expected output, and the assigned agent. The framework handles agent invocation, memory management, and inter-agent communication.

CrewAI prioritizes developer experience through clear patterns, reducing decision fatigue for standard workflows. Disadvantages include limited built-in parallelism (sequential process runs tasks serially) and manual state management for complex architectures. The framework is ideal for rapid development of known patterns.

\section{LangGraph Stateful Graphs}

LangGraph represents orchestration as explicit state machines. Agents are nodes in a directed graph; edges define state transitions and routing logic. Each node is a function that reads state, invokes an action, and returns new state. Conditions on edges enable branching: if task A succeeds, route to task B; if it fails, route to error handling\cite{LangGraph2024}.

This approach provides several advantages. State is visible and versioned, enabling precise debugging and state snapshots. Parallelism is explicit: multiple nodes can execute concurrently on independent paths that reconverge. Persistence is designed in from the start—state graphs can be serialized and resumed from checkpoints, enabling long-running workflows and failure recovery.

LangGraph's disadvantage is upfront modeling complexity. Developers must express orchestration as a state machine, which is powerful but requires architectural thinking. For straightforward sequential pipelines, the explicit graph formalism can feel over-engineered.

\section{Microsoft AutoGen}

Microsoft AutoGen approaches multi-agent orchestration through agent conversations\cite{AutoGen2023}. Agents exchange messages in round-robin or custom turn-taking order. The conversation loop continues until a termination condition (a summary is complete, a maximum turn count is reached, a user confirms the result).

AutoGen agents are defined by a system prompt (role and capabilities) and an LLM backend. Agents can invoke tools (code execution, web search, file I/O) and observe tool outputs before responding. Conversation context grows with each turn, enabling agents to reference and build on prior discussion.

This model is natural for collaborative problem-solving—multi-turn debate, iterative refinement, and consensus-seeking. It provides good observability: the conversation transcript is the audit trail. Disadvantages include sequential turn structure (parallelism is not native) and potential for conversation drift (messages grow unbounded without active summarization).

\section{Design Philosophy Trade-offs}

CrewAI's prescriptive design minimizes choice overhead but constrains customization. A researcher-writer-editor pipeline is implemented in minutes; a novel orchestration pattern may require extending the framework. LangChain and LangGraph prioritize flexibility, enabling any orchestration topology but demanding greater architectural expertise.

Sequential execution, the default in CrewAI and AutoGen, is intuitive and ensures clear ordering but limits parallelism. Wall-clock time grows with task count. LangChain and LangGraph support fine-grained parallelism through explicit chain composition and graph edges, enabling multiple tasks to run concurrently. Token consumption during parallel phases is higher, which can trigger rate limits.

Memory management reflects these philosophies. CrewAI provides built-in per-agent memory, simplifying state tracking. LangChain offers multiple memory strategies (buffer, summary, vector store), requiring selection and configuration. LangGraph's explicit state graphs provide the clearest semantics: state is visible, versioned, and persistence is designed from the start.

Tool integration differs in scope. CrewAI tools are role-specific, modeling domain workflows naturally (a Researcher has web search; a Writer has grammar checking). LangChain and LangGraph centralize tools, enabling broader reuse and composition. Each approach has merit: role-specific tools enforce clean separation; centralized tools maximize reuse.

\section{Production Readiness}

All four frameworks are production-capable. CrewAI excels at rapid deployment of standard patterns with built-in observability (callbacks, structured logging). LangChain offers the broadest ecosystem and maximum customization but requires architectural expertise. LangGraph provides the strongest guarantees around state management, determinism, and auditability.

The choice depends on requirements\cite{LangChain2024a}: for rapid development of known patterns, CrewAI; for innovation and custom workflows, LangChain; for deterministic, auditable systems with complex state, LangGraph; for collaborative multi-turn scenarios, AutoGen. For HW3, CrewAI provides a prescriptive framework ideal for research-writing-editing pipelines, while LangChain tools (web search, code execution) integrate seamlessly through CrewAI's tool system.
