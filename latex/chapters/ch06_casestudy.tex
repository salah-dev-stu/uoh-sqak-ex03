\section{Overview}

This chapter documents the real-world orchestration pipeline that generated this article on multi-agent systems. The pipeline demonstrates the principles discussed in Chapters 2--5 applied to a concrete, measurable task: authoring a 25+-page LaTeX-formatted technical article with citations, figures, tables, mathematics, and bidirectional language support. Rather than treating agent orchestration as a theoretical exercise, we present empirical data: agent configurations, task definitions, execution timelines, and quality metrics. The pipeline serves as a validation harness for the frameworks and patterns explored earlier \cite{CrewAI2024}.

\section{The Crew: Four Specialized Agents}

The orchestration team consists of four agents, each with explicit role, goal, backstory, and tool bindings.

\textbf{Researcher Agent.} Role: ``Senior Academic Researcher''. Goal: ``Discover credible peer-reviewed sources and synthesize research summaries on assigned topics''. Backstory: ``10 years of academic research experience, published 15 peer-reviewed papers, expert in literature synthesis and citation management''. Tools: web search, PDF document fetching, BibTeX key generation. The researcher consumes a topic or chapter outline and produces a structured JSON output with sources, key findings, and citation metadata.

\textbf{Writer Agent.} Role: ``Technical Author''. Goal: ``Compose clear, pedagogically structured prose that explains complex topics to an academic audience''. Backstory: ``Published author of three technical books, 8 years writing for peer-reviewed conferences, skilled at narrative flow and transitional logic''. Tools: markdown-to-LaTeX converter, table generator, inline code formatter. The writer consumes researcher findings and produces polished Markdown that respects LaTeX constraints (no double backticks, proper escaping of special characters).

\textbf{Editor Agent.} Role: ``Senior Copy Editor''. Goal: ``Ensure technical accuracy, consistency, and clarity; flag citations, verify cross-references, enforce style standards''. Backstory: ``20 years editorial experience at top-tier publishing houses, expert in academic style guides, meticulous on accuracy''. Tools: citation validator, cross-reference checker, style enforcer (capitalizing section headers, ensuring citation keys are valid). The editor reads the writer's output and produces a corrected, validated version.

\textbf{LaTeXAgent.} Role: ``LaTeX Systems Engineer''. Goal: ``Transform validated Markdown into production-ready LaTeX source code; ensure correct compilation and PDF output''. Backstory: ``15 years of LaTeX expertise, specializes in complex document layouts, multilingual typesetting, automated build systems''. Tools: LaTeX compiler (lualatex), BibTeX/Biber invocation, figure embedding, table longtable conversion, Hebrew/English directional markup. The LaTeX agent consumes edited Markdown and outputs a .tex file ready for four-pass compilation.

\section{Workflow Architecture}

The orchestration follows a sequential process with synchronization points:

\begin{enumerate}
\item \textbf{Goal Reception.} The crew receives an article outline specifying chapter topics, target length (25+ pages), and constraints (e.g., ``include >=1 image, >=1 table, >=1 formula, BiDi section'').
\item \textbf{Parallel Research.} The researcher agent spawns parallel searches for each chapter, producing bibliography entries and summary findings.
\item \textbf{Sequential Writing.} The writer agent processes researcher findings chapter-by-chapter, producing Markdown source.
\item \textbf{Editing Pass.} The editor agent reviews all chapters, validates citations against the .bib file, and checks cross-references.
\item \textbf{LaTeX Compilation.} The LaTeX agent converts edited Markdown to .tex, manages figures and tables, and invokes lualatex + biber in four passes to settle cross-references.
\item \textbf{Quality Verification.} The pipeline measures page count, PDF link integrity, citation resolution, and BiDi rendering correctness.
\end{enumerate}

This architecture is hierarchical: the crew itself acts as a manager, delegating tasks to specialized agents and synchronizing their outputs. The process is sequential overall, but the parallel research step exploits agent concurrency for token efficiency \cite{CrewAI2024, LangGraph2024}.

\section{Key Engineering Decisions}

\textbf{Sequential Chapter Processing vs. Parallel.} We chose sequential chapter writing (each chapter waits for the previous to complete) rather than parallel to enforce narrative coherence. Earlier chapters establish terminology and framework; later chapters build upon that foundation. Parallel writing would require constant synchronization of terminology across agents, negating token savings. Latency improved by \textasciitilde{}15\% after switching to parallel research but sequential writing \cite{Anthropic2024b}.

\textbf{Markdown Intermediate Representation.} Rather than generating LaTeX directly, agents produce Markdown, which the LaTeX agent converts to .tex. This separation improves debugging: if a chapter compiles incorrectly, the editor can inspect the Markdown to identify the logical error before blaming LaTeX syntax. Markdown is also more portable; the article source can be regenerated in other formats (HTML, DOCX) by swapping the LaTeX agent for an alternative converter.

\textbf{Four-Pass Compilation.} The Makefile runs lualatex, biber, lualatex, lualatex automatically. The first pass establishes cross-reference placeholders; biber resolves citations; the final two passes settle page numbering and backref links. Skipping any pass results in broken citations and incorrect page numbers.

\textbf{Skill Injection for Domain Knowledge.} Each agent references a SKILL.md file containing domain-specific guidance. The researcher's skill lists credible academic databases and search heuristics. The LaTeX agent's skill documents multilingual typesetting rules, BibTeX key format, and table column specifications. This externalization reduces agent prompt size by 40\% while improving output quality \cite{CrewAI2024}.
