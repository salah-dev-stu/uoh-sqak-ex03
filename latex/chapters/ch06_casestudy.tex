\chapter{Case Study: This Article's Production Pipeline}\label{ch:casestudy}

\section{Overview}

This case study demonstrates the orchestration patterns discussed throughout this article applied to a real-world problem: automatically generating a peer-reviewed technical article using a CrewAI-based multi-agent system. The pipeline exemplifies sequential orchestration with quality gates, token budget management, and structured output validation \cite{CrewAI2024}.

Manual technical article writing is slow and expensive. A senior researcher might spend 20 hours researching a topic, 15 hours drafting, 10 hours editing, and 5 hours formatting—50 hours total. Multi-agent orchestration compresses this timeline while reducing bottlenecks: parallel research, concurrent editing passes, and automated \LaTeX{} production. The target is a polished 15-page PDF on a chosen technical topic, complete with citations, figures, and proper formatting \cite{Anthropic2024b}.

\section{The Crew: Four Specialized Agents}

The system comprises four specialized agents, each with distinct expertise and responsibilities:

\begin{enumerate}
\item \textbf{Researcher Agent} gathers factual information from web sources, compiles citations, and validates claims. The researcher operates within a token budget of 12,000 tokens per invocation and produces structured notes in JSON format with source attribution.

\item \textbf{Writer Agent} produces prose from research notes, organizing content into chapters with proper narrative flow and internal cross-references. The writer agent consumes approximately 15,000 tokens and outputs Markdown-formatted chapter content ready for editing review.

\item \textbf{Editor Agent} refines language, checks consistency, flags missing citations, and ensures structural coherence across chapters. The editor agent operates within 8,000 tokens and validates all cross-references before handoff downstream.

\item \textbf{LaTeX Producer Agent} converts edited Markdown into polished \LaTeX{}, embedding figures, formatting tables, and generating cross-references automatically. The \LaTeX{} agent consumes approximately 6,000 tokens and produces compilation-ready \texttt{.tex} files.
\end{enumerate}

Supporting these primary agents are two critical infrastructure components: a \textbf{Gatekeeper} that enforces token budgets (max 50,000 tokens per agent per run) and rate-limits API calls to prevent service degradation, and a \textbf{Supervisor Agent} that validates each agent's output against quality criteria before passing it downstream. A \textbf{Skill Layer} injects domain-specific expertise: the Researcher agent accesses academic paper databases; the Writer agent references writing style guides; the \LaTeX{} Producer knows typography and bidirectional encoding \cite{LangChain2024a}.

\section{Workflow Architecture}

A sequential pipeline orchestration ensures each agent produces output suitable for the next stage. The Researcher agent completes, handing off structured notes to the Writer. The Writer produces Markdown chapters, which flow to the Editor. The Editor refines and validates citations, then passes to the \LaTeX{} Producer. Finally, the \LaTeX{} Producer compiles a 15-page PDF with proper formatting, embedded figures (Python-generated charts), and linked bibliography.

\section{Key Engineering Decisions}

Several critical decisions shaped the pipeline architecture. First, token budgets are enforced via JSON configuration files, never hardcoded in Python source. This enables rapid iteration without recompilation. Second, the Gatekeeper pattern centralizes all external API calls—LLM invocations, web searches, and \LaTeX{} compilation—through a single rate-limited interface. This prevents cascading failures and simplifies monitoring. Third, output validation occurs at each stage: the Researcher's JSON must parse; the Writer's Markdown must conform to a schema; the Editor's citations must resolve to the bibliography; the \LaTeX{} Producer's output must compile cleanly. These gates prevent error propagation downstream and ensure quality at every handoff \cite{CrewAI2024}.

The case study demonstrates that multi-agent orchestration excels when tasks are naturally decomposable, when specialized expertise improves output quality, and when quality gates prevent error propagation. Total token consumption per run averages approximately 41,000 tokens, comfortably under the 50,000-token budget. Wall-clock latency is approximately 8 minutes in sequential mode: 2 minutes research, 2 minutes writing, 1.5 minutes editing, 1 minute \LaTeX{} compilation. Without orchestration, a single agent attempting the entire task would require 60,000 or more tokens and 15 or more minutes of wall-clock time, with lower output quality and greater risk of hallucination. This pattern generalizes to code generation, market analysis, product documentation, and scientific research—any domain where structured, multi-stage workflows add demonstrable value.
